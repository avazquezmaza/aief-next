# OpenSpec Workflow with AIEF

## AIEF-native workflow

```text
Define -> Design -> Build -> Verify -> Evidence
```

## OpenSpec-enhanced workflow

```text
Define
  -> Create OpenSpec proposal
  -> Add spec deltas
  -> Create task list
  -> Review proposal
  -> Build
  -> Verify
  -> Capture AIEF evidence
```

## Human and AI Responsibilities

Human:

- approves scope,
- approves proposal,
- approves release.

AI:

- drafts proposal,
- drafts spec changes,
- creates tasks,
- implements scoped changes,
- updates evidence.

## Important Rule

OpenSpec may help define the change, but AIEF still requires evidence before work is considered complete.
