# Specification

## Goal
Make AIEF easier to understand and adopt by providing a clear navigation layer.

## Requirements
- Users must know what to read first.
- Users must know what to copy for a new project.
- Users must know how to adopt AIEF in an existing project.
- Users must know what the AI assistant uses.
- Users must know what they should edit.
- Users must know what they should avoid changing.
- Users must have Windows, Linux, and macOS setup paths.
- Users must understand how OpenSpec and Specboot fit into AIEF.
- Documentation must stay concise and practical.

## Acceptance Criteria
- [ ] `docs/navigator/README.md` exists.
- [ ] `docs/navigator/decision-tree.md` exists.
- [ ] `docs/navigator/new-project.md` exists.
- [ ] `docs/navigator/existing-project.md` exists.
- [ ] `docs/navigator/tooling.md` exists.
- [ ] `docs/navigator/ai-assistants.md` exists.
- [ ] `docs/navigator/workflows.md` exists.
- [ ] OS-specific install guides exist.
- [ ] Mermaid diagrams exist.
- [ ] The Navigator links to existing AIEF docs and examples.
