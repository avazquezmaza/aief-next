# Specification

## Goal

Make the repository easier to navigate and cleaner for first-time users.

## Requirements

- The repository should have only one Codex instruction file.
- The repository should expose the Navigator from the root.
- Documentation should have a central index.
- Release notes should align with the current MVP maturity level.
- The cleanup should not introduce new features.

## Acceptance Criteria

- [ ] Only `CODEX.md` exists.
- [ ] `codex.md` is removed.
- [ ] `NAVIGATOR.md` exists at the repository root.
- [ ] `docs/index.md` exists.
- [ ] `releases/v0.1.0.md` exists.
- [ ] No `demo-project/` is committed.
