# Specification

## Goal

Create a practical starter project that demonstrates the minimum AIEF structure for a real project.

## Requirements

- The starter project must be copy-ready.
- The starter project must include `README.md`.
- The starter project must include `AGENTS.md`.
- The starter project must include `CLAUDE.md`, `GEMINI.md`, `CODEX.md`, and `CURSOR.md`.
- The starter project must include:
  - `changes/`
  - `knowledge/`
  - `src/`
  - `tests/`
  - `docs/`
- The starter project must include one example completed Change.
- The starter project must keep instructions short and practical.

## Acceptance Criteria

- [ ] `starter-project/README.md` explains how to use the starter.
- [ ] `starter-project/AGENTS.md` defines project-level agent rules.
- [ ] Assistant-specific files exist and refer back to `AGENTS.md`.
- [ ] `starter-project/changes/README.md` explains how to create Changes.
- [ ] `starter-project/knowledge/README.md` explains what knowledge captures.
- [ ] `starter-project/src/README.md` explains where implementation code goes.
- [ ] `starter-project/tests/README.md` explains where tests go.
- [ ] One example Change exists under `starter-project/changes/0000-example-change/`.
