# Specification

## Goal

A first-time user adopting a project sees no false alarms, no machine-generated homework, and always knows the next step.

## Requirements

- Adoption evidence includes: created files/folders/standards, detected signals with reasons, recommended Skills with reasons, and the guarantees (no functional code changed, no existing files overwritten, only AIEF structure created), under the marker "Generated by AIEF during adoption." Change files remain the only source of truth.
- Adoption tasks reflect reality: machine-done items checked; human items (adapt standards, verify, close) unchecked.
- `verify` Next logic: FAIL → fix and re-run; PASS with no open Change → honest "no open Change" hint; PASS with placeholder-evidence active Change → prompt then close; PASS with ready active Change → close --yes.
- `status` separates required (✓/!) from optional (`·` + "(optional)") checks.
- `doctor` environment shows openspec as optional in both states.

## Acceptance Criteria

- [ ] Adoption evidence auto-generated, non-placeholder, listing artifacts, signals, skills and guarantees (test).
- [ ] `verify` prints `Next:` on PASS and FAIL (test).
- [ ] `doctor` on an adopted project shows no `!` for Navigator/Profiles/adapters and shows openspec as optional (test).
- [ ] Adopt idempotence and app-code-untouched tests still pass.
- [ ] Full suite passes; `aief verify` passes; Change closed via `aief close --yes`.
