# Workflow Diagram

```mermaid
flowchart LR
    A[Define] --> B[Design]
    B --> C[Build]
    C --> D[Verify]
    D --> E[Evidence]
    E --> F[Release]
```

## Meaning

- Define: clarify the Change.
- Design: decide how to implement.
- Build: make the change.
- Verify: check acceptance criteria.
- Evidence: capture proof.
- Release: approve and publish.
