# Project Lifecycle

AIEF projects evolve through Changes.

```text
Project
  -> Change
  -> Evidence
  -> Knowledge
  -> Next Change
```

## Project

The project holds context.

Examples:

- README
- AGENTS
- docs
- knowledge

## Change

A Change is the unit of work.

Examples:

- feature
- bug fix
- refactor
- research spike
- documentation update

## Evidence

Evidence proves what happened.

Examples:

- test results
- manual verification
- review notes
- known issues

## Knowledge

Knowledge captures what should be remembered.

Examples:

- decisions
- lessons learned
- constraints
- patterns

## Loop

Every completed Change should make the project easier to understand.
