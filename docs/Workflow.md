# AIEF Workflow

AIEF uses a simple workflow:

```text
Define -> Design -> Build -> Verify -> Release
```

## Define

Clarify the goal, scope, and success criteria.

Primary file:

```text
change.md
```

## Design

Decide how the change will be implemented.

For small changes, this can be lightweight.

Primary files:

```text
spec.md
tasks.md
```

## Build

Implement the requested change.

The implementation should stay within the Change scope.

## Verify

Confirm that requirements and acceptance criteria are met.

Verification can be automated or manual.

## Release

Capture evidence, known issues, and final notes.

Primary file:

```text
evidence.md
```
