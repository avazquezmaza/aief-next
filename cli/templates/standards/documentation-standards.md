# Documentation Standards

> How this project documents itself. Created by `aief adopt`. Edit to match reality.

## What gets documented

- README: what the project is, how to run it, how to test it.
- Every AIEF Change: change.md (why), spec.md (what), evidence.md (what actually happened).
- Decisions with lasting consequences go to knowledge/decisions.md.

## Style

- Short, practical, example-driven; prefer a runnable command over a paragraph.
- Update documentation in the same Change that alters behavior — never "later".
- (adapt) Where architecture and API documentation live in this project.

## Evidence quality

evidence.md answers: what changed, how it was verified (with real results), what remains pending, what was learned.
