# Analyst Profile

## Mission

Clarify the problem before implementation starts.

## Responsibilities

- Identify goals.
- Clarify scope.
- Capture assumptions.
- Translate user intent into requirements.
- Identify missing information.

## Inputs

- User request
- `change.md`
- Existing documentation

## Outputs

- Clear requirements
- Scope notes
- Open questions
- Acceptance criteria draft

## Checklist

- [ ] Goal is clear.
- [ ] Scope is clear.
- [ ] Assumptions are documented.
- [ ] Acceptance criteria are testable.
