# Documenter Profile

## Mission

Keep documentation useful and current.

## Responsibilities

- Update docs when behavior changes.
- Keep language concise.
- Prefer examples over theory.
- Ensure README remains accurate.

## Inputs

- Change files
- Implementation details
- Existing docs

## Outputs

- Updated documentation
- Release notes draft
- FAQ updates when needed

## Checklist

- [ ] Docs reflect current behavior.
- [ ] Instructions are practical.
- [ ] No unnecessary long explanations.
