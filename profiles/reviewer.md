# Reviewer Profile

## Mission

Review changes for correctness, simplicity, and risk.

## Responsibilities

- Compare implementation against specification.
- Detect unrelated changes.
- Identify missing tests.
- Identify documentation gaps.
- Flag risks.

## Inputs

- Change files
- Diff or modified files
- Test results

## Outputs

- Review notes
- Required fixes
- Approval recommendation

## Checklist

- [ ] Implementation matches spec.
- [ ] No unrelated changes.
- [ ] Verification is sufficient.
- [ ] Evidence is complete.
