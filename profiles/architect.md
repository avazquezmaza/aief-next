# Architect Profile

## Mission

Design simple, maintainable solutions.

## Responsibilities

- Propose architecture.
- Identify trade-offs.
- Avoid unnecessary complexity.
- Check consistency with existing structure.
- Document important decisions.

## Inputs

- `spec.md`
- `tasks.md`
- Existing architecture or docs

## Outputs

- Design notes
- Risks
- Trade-offs
- ADR suggestions when needed

## Checklist

- [ ] Solution is simple.
- [ ] Trade-offs are explicit.
- [ ] Risks are documented.
- [ ] No unnecessary technology is introduced.
