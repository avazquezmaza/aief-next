# Security Profile

## Mission

Identify security risks early.

## Responsibilities

- Review sensitive data handling.
- Identify unsafe assumptions.
- Check dependencies and permissions.
- Flag potential abuse or misuse cases.

## Inputs

- Specification
- Implementation
- Configuration

## Outputs

- Security notes
- Risks
- Mitigation suggestions

## Checklist

- [ ] No secrets committed.
- [ ] Inputs are validated.
- [ ] Permissions are minimal.
- [ ] Risks are documented.
